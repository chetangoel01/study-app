# Study App

A full-stack study planner for interview prep.

The current app uses:

- `client/`: React + Vite SPA
- `server/`: Hono + SQLite backend
- `build_study_data.py`: curated curriculum generator for `curriculum.json`
- `pipeline/`: optional enrichment pipeline for `knowledge-base.json`

The app organizes the curriculum into four tracks, persists progress per user account, supports notes and module-level content, and can enrich modules from the knowledge graph generated by the Python pipeline.

## Local development

Start the backend:

```bash
npm run dev:server
```

Start the frontend:

```bash
npm run dev:client
```

Then open [http://localhost:5173](http://localhost:5173).

## Test and build

Run the full test suite:

```bash
npm test
```

Build both the server and client:

```bash
npm run build
```

Run the compiled production server:

```bash
npm start
```

By default the production app is served at [http://localhost:3000](http://localhost:3000).

## Refresh curriculum data

If you change the curated module definitions, regenerate `curriculum.json`:

```bash
python3 build_study_data.py
```

## Run the enrichment pipeline

Install the optional Python dependencies:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r pipeline/requirements.txt
```

Run the pipeline:

```bash
LLM_API_KEY=... python3 pipeline/runner.py
```

Or, if you use OpenRouter-compatible credentials directly:

```bash
OPENROUTER_API_KEY=... python3 pipeline/runner.py
```

Both variants write `knowledge-base.json`, which the server loads at startup.

## Pull a purchased LeetCode course

If you want to bring a purchased LeetCode Explore course into this repo as a local source artifact:

1. Export your authenticated LeetCode cookies to a local file.
2. Save that file somewhere local, outside git tracking.
3. Run:

```bash
python3 download_leetcode_course.py --cookies /path/to/leetcode.cookies.txt
```

This writes local artifacts under `course_exports/leetcode-crash-course/`.

Treat the cookie export like a password and do not commit it. If `course_exports/leetcode-crash-course/manifest.json` is present, `build_study_data.py` imports the selected course chapters into the generated curriculum, and the pipeline can reuse the exported item JSON without re-authenticating against LeetCode.
