Consider the following pseudocode: stack = [] for (num : nums_array) { while (stack not empty AND num <= stack.top()) { stack.pop() } stack.push(num) } The following is a list of questions for you to answer. Click on each question to reveal the answer. What kind of data structure is this? How does the code implement it? This code implements a monotonically increasing stack. Before we push an element num , we remove all elements in the stack that are greater than num . Thus, by the time we push num onto the stack, num is the largest element in the stack. As num will be at the top (the final element), stack is increasing. What is the time complexity of this code, where n is the length of nums_array? Consider that there is a nested while loop. The time complexity is O(n). Each iteration of the while loop pops an element from the stack, and each element can only be pushed to the stack once. Thus, the while loop never runs more than n times in total. Thus, each for loop iteration costs amortized O(1). While any given for loop iteration could be very expensive, the iterations average out to constant time when you consider the entire algorithm.
