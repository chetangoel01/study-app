Another common pattern is problems that involve intervals [start, end] . Usually, these problems can be solved by first sorting the input, then iterating over it while comparing adjacent intervals. Let's look at some examples. Example 1: 252. Meeting Rooms Given an array of meeting times intervals where intervals[i] = [start, end] indicates the meeting runs from [start, end) , determine if one person could attend all meetings. For example, given intervals = [[0, 30], [5, 10], [15, 20]] , return false . If you attend the [0, 30] meeting, then you cannot attend the other two. The brute force approach would be to look at every pair of meetings and check if there is overlap, which would take , where is the number of meetings. We can improve on this by observing that if there is a conflict, then the conflicting meetings would be adjacent if we sorted the input. If we sort the meetings by their start time, then the meetings are in the order in which they should be attended. If the meeting starts before the meeting ends, then there is a conflict because you would need to leave the meeting early to be on time for the meeting. Merge Intervals Given an array of intervals where intervals[i] = [start, end] , merge all overlapping intervals, and return an array of the non-overlapping intervals that cover all the intervals in the input. For example, given intervals = [[1, 3], [2, 6], [8, 10], [15, 18]] , return [[[1, 6], [8, 10], [15, 18]]] . The first two intervals merge to form [1, 6] , and then the next two intervals have no overlap. In the previous problem, we sorted the intervals by their start times, and then compared adjacent intervals to check for overlap. We can use the same process here. Let's say that we have two intervals that overlap - [a, b] and [c, d] . The merged interval should be [min(a, c), max(b, d)] . Let's sort the input by start times and build the answer array ans one interval at a time. At any given interval, if we find that start is less than or equal to the previous interval's end time, there is an overlap and we should merge the intervals. Since we sort by start time, the previous interval's start will always be smaller, so we can ignore the min(a, c) part from the previous paragraph. We just need to modify the new end with max(b, d) .
