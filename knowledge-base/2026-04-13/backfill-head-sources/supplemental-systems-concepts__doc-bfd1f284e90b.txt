Concurrency pattern
Appearance
A concurrency pattern is a software design pattern that supports concurrent processing.
Examples
[edit]Examples include:
- Active object[1][2]
- Balking pattern
- Barrier
- Double-checked locking
- Guarded suspension
- Monitor object
- Nuclear reaction
- Reactor pattern
- Readers–writer lock
- Scheduler pattern
- Thread pool pattern
- Thread-local storage
